Unit 1 · 6 hrs
Data Communication & Network Basics
Network topologies, OSI model deep dive, transmission media (guided vs unguided), LAN/WAN architectures, and virtual LAN concepts.

1.1 Introduction to Data Communication

Data communication involves the transfer of digital data between two or more nodes through a transmission medium. The effectiveness depends on four fundamental characteristics: Delivery, Accuracy, Timeliness, and Jitter.

Components of Data Communication:
  • Message: The information (data) to be communicated (text, numbers, pictures, audio).
  • Sender & Receiver: The devices initiating and accepting the transmission.
  • Transmission Medium: The physical path by which a message travels (Twisted-pair wire, Coaxial cable, Fiber-optic, Radio waves).
  • Protocol: A set of rules that govern data communications. Without protocols, devices may be connected but cannot understand each other.

1.2 Network Topologies Visualized

The physical or logical layout of a network. Choosing a topology impacts cost, scalability, and fault tolerance.

TopologyProsCons
StarEasy to install, robust, fault isolationSingle point of failure (Hub/Switch)
MeshNo traffic issues, extremely robust & secureExpensive, difficult installation, huge cabling
BusEasy to connect, requires less cableDifficult fault isolation, hard to scale
RingEqual access, orderly networkOne node failure can break the ring

1.3 Transmission Media

Media can be categorized into Guided (wired) and Unguided (wireless).

  • Twisted Pair: STP (Shielded) and UTP (Unshielded). Common in LANs (Cat5e, Cat6). Uses RJ-45 connectors.
  • Coaxial Cable: Higher frequency range than twisted pair. Used in Cable TV.
  • Fiber-Optic: Transmits signals via light (photons) instead of electrons. Uses Total Internal Reflection. Immune to Electromagnetic Interference (EMI).
  • Unguided Media: Radio waves (multicast, omnidirectional), Microwaves (unicast, line-of-sight), and Infrared (short-range, line-of-sight).

1.4 OSI Model Recap & Layer Interactions

Encapsulation: As data moves from layer 7 down to layer 1, each layer adds its own header (and sometimes trailer). This is encapsulation. Decapsulation happens at the receiver end.

Protocol Data Units (PDUs):
- Layers 5-7 (App, Pres, Sess): Data / Message
- Layer 4 (Transport): Segment (TCP) or Datagram (UDP)
- Layer 3 (Network): Packet
- Layer 2 (Data Link): Frame
- Layer 1 (Physical): Bits
Unit 2 · 6 hrs
Bandwidth Utilization & Multiplexing
FDM, TDM, WDM techniques, Spread Spectrum (DSSS & FHSS), and efficient channel usage.

2.1 Multiplexing Overview

Multiplexing allows the simultaneous transmission of multiple signals across a single data link. A Multiplexer (MUX) combines the signals, and a Demultiplexer (DEMUX) separates them at the destination.

TypeDescriptionDomain
FDM (Frequency Division)Assigns non-overlapping frequency ranges to different signals. Used in Radio/TV.Analog
WDM (Wavelength Division)Combines different colors of light onto a single optical fiber.Analog/Optical
TDM (Time Division)Allocates alternating time slots to signals. Synchronous TDM assigns empty slots; Statistical TDM skips empty ones.Digital

2.2 Spread Spectrum Communication

Spread spectrum techniques spread the bandwidth needed to transmit data over a much wider range than strictly necessary. This prevents jamming, eavesdropping, and fading.

FHSS (Frequency Hopping Spread Spectrum):
The signal rapidly hops among many different frequencies in a pseudo-random sequence known to both sender and receiver. Used in original Bluetooth.

DSSS (Direct Sequence Spread Spectrum):
The original data signal is multiplied by a "chipping code" (a high-speed sequence of bits). This spreads the signal across a wide frequency band. Used in 802.11b Wi-Fi.
Nyquist Bit Rate vs Shannon Capacity:
  • Nyquist (Noiseless Channel): BitRate = 2 * Bandwidth * log2(L), where L is the number of signal levels.
  • Shannon (Noisy Channel): Capacity = Bandwidth * log2(1 + SNR), where SNR is the Signal-to-Noise Ratio.
Unit 3 · 8 hrs
Data Link & MAC Sub-Layer Deep Dive
Error detection/correction (CRC, Hamming), flow/error control (ARQ protocols), MAC algorithms (ALOHA, CSMA/CD, CSMA/CA).

3.1 Error Detection and Correction

Data Link layer guarantees reliable node-to-node delivery, detecting errors introduced by the physical layer.

  • Parity Check: Adds a single bit to make the number of 1s even (or odd). Only detects single-bit errors.
  • Checksum: Used primarily by TCP/IP. The sender adds segments using 1's complement arithmetic and sends the complement as the checksum.
  • Cyclic Redundancy Check (CRC): A powerful polynomial division method. The generator polynomial (e.g., CRC-32) is XOR-divided against the data block. The remainder is appended as the FCS (Frame Check Sequence).
  • Hamming Code: An error-correction code that uses redundant bits to not only detect but locate single-bit errors. Redundant bits `r` must satisfy `2^r >= m + r + 1` (where m is data bits).

3.2 Flow Control: Sliding Window Protocols

Flow control ensures a fast sender does not overwhelm a slow receiver.

ProtocolSender WindowReceiver WindowRetransmission
Stop-and-Wait11Retransmits 1 frame upon timeout
Go-Back-N2^m - 11Retransmits the lost frame and ALL subsequent frames
Selective Repeat2^(m-1)2^(m-1)Retransmits ONLY the specifically lost frame. Requires sorting.

3.3 Multiple Access Protocols (MAC)

When multiple nodes share a broadcast link, collision happens. MAC protocols coordinate access.

ALOHA: Pure ALOHA allows transmission anytime (Max Efficiency = 18.4%). Slotted ALOHA restricts transmission to slot boundaries (Max Efficiency = 36.8%).

CSMA/CD (Collision Detection): Standard for wired Ethernet (802.3). Nodes listen before talking. If a collision is detected during transmission, they immediately stop, send a JAM signal, and wait a random backoff time (Binary Exponential Backoff).

CSMA/CA (Collision Avoidance): Standard for Wireless (802.11). Wireless nodes can't easily detect collisions (Hidden Node Problem). They use RTS (Request to Send) and CTS (Clear to Send) packets to reserve the channel before transmitting data.

Network Layer & Routing Protocols

4.1 Internet Protocol (IPv4 vs IPv6)

The Network layer is responsible for logical addressing and routing. IPv4 uses 32-bit addresses, divided into 5 classes (A, B, C, D, E). IPv6 uses 128-bit hexadecimal addressing to overcome IPv4 exhaustion.

4.2 Subnetting & CIDR

Subnetting borrows bits from the Host ID to create a Subnet ID. CIDR (Classless Inter-Domain Routing) replaces classes completely with a suffix (e.g., /24).

Subnetting Math:
Given `192.168.1.0/26`
- Borrowed bits = 26 - 24 = 2.
- Number of Subnets = `2^2 = 4`.
- Host bits remaining = 32 - 26 = 6.
- Hosts per Subnet = `2^6 - 2 = 62` (Subtracting Network & Broadcast ID).

4.3 Auxiliary Protocols (ARP & DHCP)

DHCP DORA Process:
  • Discover: Client broadcasts to find a DHCP Server.
  • Offer: Server responds with a unicast offering an IP.
  • Request: Client broadcasts its acceptance of the IP.
  • Acknowledge: Server formally confirms the IP lease.
ARP (Address Resolution Protocol): Resolves logical IPv4 addresses to physical MAC addresses.

4.4 Routing Protocols

  • RIP (Routing Information Protocol): Distance Vector protocol using Bellman-Ford algorithm. Uses hop count (max 15). Sends entire routing table to neighbors.
  • OSPF (Open Shortest Path First): Link State protocol using Dijkstra's algorithm. Uses cost (bandwidth). Floods neighbor status to all routers.
  • BGP (Border Gateway Protocol): Path Vector protocol used to route traffic between Autonomous Systems on the internet.

5.3 TCP Congestion Control

To prevent the network from collapsing, TCP controls the transmission window size `cwnd`.

  • Slow Start: The `cwnd` starts at 1 MSS (Maximum Segment Size) and grows exponentially (doubles every RTT).
  • Congestion Avoidance: Once `cwnd` reaches a threshold (`ssthresh`), the window grows linearly (adds 1 MSS per RTT) to probe for bandwidth carefully.
  • Congestion Detection (Timeout): If a timeout occurs, severe congestion is assumed. `ssthresh` is halved, and `cwnd` is reset back to 1. Slow start begins again.
  • Fast Recovery: If 3 duplicate ACKs are received, moderate congestion is assumed. `ssthresh` is halved, `cwnd` is set to the new `ssthresh`, and linear growth begins (skips slow start).
Unit 6 · 7 hrs
Application Layer & Network Security
DNS, TELNET, EMAIL (SMTP/POP3/IMAP), FTP, HTTP, SNMP, Bluetooth, Firewalls, and Cryptography.

6.1 Core Internet Protocols

The Application layer provides network services directly to end-user applications.

Domain Name System (DNS): Acts as the phonebook of the internet. Resolves human-readable domain names (www.google.com) to IP addresses. Operates over UDP Port 53 for queries, TCP Port 53 for zone transfers.

HTTP / HTTPS: Hypertext Transfer Protocol. Stateless protocol running over TCP Port 80. HTTPS (Port 443) uses TLS/SSL to encrypt traffic.

FTP (File Transfer Protocol): Uses two ports. Port 21 for Control (commands) and Port 20 for Data transfer. Active and Passive modes exist.

6.2 Electronic Mail Architecture

Email relies on a combination of push and pull protocols.

  • SMTP (Port 25/587): Simple Mail Transfer Protocol. A Push protocol used to send emails from a client to a server, or between servers.
  • IMAP (Port 143/993): Internet Message Access Protocol. A Pull protocol used to read emails, syncing read state and folders across multiple devices.
  • POP3 (Port 110/995): Post Office Protocol. A Pull protocol used to download emails to a local device, traditionally deleting them from the server afterward.

6.3 Network Security & Cryptography

Security goals are often defined by the CIA Triad: Confidentiality, Integrity, and Availability.

Symmetric vs Asymmetric Key Cryptography:
  • Symmetric Key: The same key is used for both encryption and decryption (e.g., DES, AES). Very fast, but key distribution is challenging.
  • Asymmetric Key (Public Key): Uses a pair of keys: a Public Key (shared) and a Private Key (secret). E.g., RSA. If Alice wants to send a secret to Bob, she encrypts it with Bob's Public Key. Only Bob's Private Key can decrypt it.
Firewalls: Hardware or software that inspects incoming and outgoing traffic based on rules. Packet Filtering Firewalls inspect Layer 3/4 headers. Application Gateways (Proxies) inspect Layer 7 payload.

Routing Protocols Deep Dive

Distance Vector Routing (RIP)

  • Uses Bellman-Ford Algorithm.
  • Routers share their entire routing table with neighbors only.
  • Suffers from Count-to-Infinity problem.
  • Solution: Split Horizon, Route Poisoning, Hold-down timers.
  • Metric: Hop Count (Max 15 hops).

Link State Routing (OSPF)

  • Uses Dijkstra's Algorithm for Shortest Path.
  • Routers share information about their direct neighbors with the entire network via Flooding.
  • Every router builds a complete topological map of the network.
  • Metric: Cost (Bandwidth-based). Faster convergence than RIP.

Unit 1 · 7 hrs
Lexical Analysis & Automata
Regular languages, finite automata (DFA/NFA), RE→FA conversion algorithms, Thompson's Construction, Subset Construction.

1.1 Role of Lexical Analyzer (Scanner)

The first phase of a compiler. It reads the source code character by character, strips out whitespace and comments, and groups characters into meaningful sequences called Tokens.

Terminology:
  • Token: An abstract category representing a unit of syntax (e.g., `id`, `num`, `keyword_if`).
  • Lexeme: The actual sequence of characters matching a token (e.g., `count`, `42`, `if`).
  • Pattern: The rule (defined via Regular Expressions) describing how a token is formed.

1.2 Finite Automata (FA)

Regular Expressions are implemented using Finite Automata. NFA (Nondeterministic FA) allows multiple transitions for the same symbol and `ε` (empty) transitions. DFA (Deterministic FA) allows exactly one transition per symbol.

1.3 Automata Conversion pipeline

  • Thompson's Construction (RE → NFA): Systematically builds an NFA from a Regular Expression using simple templates for union (`|`), concatenation, and Kleene closure (`*`). Generates many `ε` transitions.
  • Subset Construction (NFA → DFA): Converts NFA to DFA. Each state in the DFA represents a set of states in the NFA (the epsilon-closure). Worst-case complexity is exponential `O(2^N)`.
  • DFA Minimization (Hopcroft's Algorithm): Partitions DFA states into equivalent classes based on distinguishable behaviors. Reduces the automaton to its minimal state representation.
Unit 2 · 9 hrs
Syntax Analysis & Parsing Algorithms
Context-Free Grammars (CFGs), Top-down parsers (LL(1)), Bottom-up parsers (LR(0), SLR, LALR, CLR), Ambiguity.

2.1 Context-Free Grammars (CFG)

A parser receives tokens and determines if they form a valid syntax tree. A CFG is defined as `G = (V, T, P, S)` where V=Variables, T=Terminals, P=Productions, S=Start Symbol.

Grammar Transformations:
  • Left Recursion Removal: A production `A → Aα | β` causes infinite loops in top-down parsers. Convert to right recursion: `A → βA'` and `A' → αA' | ε`.
  • Left Factoring: Resolves FIRST set collisions `A → αβ1 | αβ2`. Transformed to `A → αA'` and `A' → β1 | β2`.

2.2 Top-Down vs Bottom-Up Parsing

Top-Down parsing builds the parse tree from the root (Start symbol) to leaves. Bottom-up parsing builds it from leaves to the root via Shift and Reduce operations.

Parser TypeClassPower & ComplexityKey Logic
LL(1)Top-DownLow Power. No Left Recursion allowed.Predictive parsing using FIRST() and FOLLOW() sets. Needs Left Factoring.
LR(0)Bottom-UpVery Basic. Suffers from Shift/Reduce conflicts.Uses automaton of LR(0) items (`A → α . β`).
SLR(1)Bottom-UpMedium Power. Solves basic conflicts.Only reduces `A → α .` if the lookahead token is in FOLLOW(A).
LALR(1)Bottom-UpHigh Power. Industry Standard (YACC/Bison).Merges LR(1) states that have the same core, drastically reducing state count while maintaining power.
CLR(1)Bottom-UpMaximum Power.Canonical LR(1). Uses exact lookahead items (`A → α . β, c`). Generates massive parsing tables.

2.3 Ambiguity and Dangling-Else

A grammar is Ambiguous if it can produce more than one Leftmost Derivation (or Parse Tree) for the same string.

The Dangling-Else Problem:
Given `if E1 then if E2 then S1 else S2`. To which `if` does the `else` belong?
Solution: The standard rule in most programming languages is to match an `else` with the closest unmatched `if`.
Unit 3 · 8 hrs
Semantic Analysis & Type Checking
Syntax-Directed Translation (SDT), Attribute grammars, symbol table management, and runtime environments.

3.1 Role of Semantic Analyzer

While the Syntax Analyzer ensures the code structure is correct, the Semantic Analyzer ensures it makes logical sense (e.g., verifying a variable is declared before use, or ensuring `int + float` is handled correctly via type coercion).

3.2 Syntax-Directed Translation (SDT)

SDT attaches semantic rules to the productions of a Context-Free Grammar. It uses Attributes to pass values up and down the parse tree.

Attribute TypeComputation DirectionGrammar Class
SynthesizedComputed from Children → ParentS-Attributed (Evaluated Bottom-Up)
InheritedComputed from Parent/Siblings → NodeL-Attributed (Evaluated Top-Down, Left-to-Right)

3.3 The Symbol Table

Symbol Table Management:
A crucial data structure maintained across all compiler phases. It stores metadata about identifiers (Type, Scope, Memory Location, Array Dimensions).

Implementation: Usually implemented using a Hash Table for O(1) average lookup time. Scope is often handled using a stack of hash tables (one table per scope block).

3.4 Activation Records

During execution, the Runtime Environment allocates an Activation Record (Stack Frame) for every function call. It stores local variables, return addresses, static links, and saved machine status.

Unit 4 · 7 hrs
Intermediate Code & Optimization
Three-Address Code (TAC), Control-flow analysis, DAGs, and Machine-Independent Optimisations.

4.1 Three-Address Code (TAC)

TAC is an abstract, machine-independent representation. Each instruction has at most one operator and three operands. It is represented in memory using arrays of Quadruples (op, arg1, arg2, result) or Triples.

Source: a = b * -c + b * -c
TAC representation:
t1 = -c
t2 = b * t1
t3 = -c
t4 = b * t3
t5 = t2 + t4
a = t5

4.2 Machine-Independent Optimizations

These optimizations improve code efficiency without caring about the underlying CPU architecture.

  • Constant Folding: Evaluating expressions with constants at compile time. x = 2 * 3.14 → x = 6.28.
  • Constant Propagation: Substituting the value of a constant variable wherever it is used.
  • Dead Code Elimination: Removing unreachable code (e.g., code after a return statement) or variables that are never read.
  • Common Subexpression Elimination: If x = a * b and later y = a * b, the compiler reuses `x`. Implemented efficiently using Directed Acyclic Graphs (DAGs).

4.3 Loop Optimizations

Loops consume the most execution time, so optimizing them is highly effective.

  • Code Motion: Moving loop-invariant computations outside the loop body.
  • Strength Reduction: Replacing expensive operations (multiplication) with cheaper ones (addition) inside loops.
  • Loop Unrolling: Replicating the loop body to reduce the branch-checking overhead.
Unit 5 · 4 hrs
Architecture-Dependent Optimisation
Instruction scheduling for pipelines, cache optimization, and register allocation (graph coloring).

5.1 Target Machine Code Generation

The final phase maps Intermediate Code to specific CPU Assembly instructions. It involves instruction selection, register allocation, and instruction scheduling.

Register Allocation via Graph Coloring:
Registers are the fastest and most scarce memory in a CPU. Variables are represented as nodes in an Interference Graph. If two variables are "live" at the same time, an edge connects them. The graph is colored using k colors (where k = number of CPU registers). If a node cannot be colored, we must Spill the variable into RAM (stack memory), which adds slow load/store instructions.

5.2 Peephole Optimization

A simple, localized optimization technique performed on the final target code.

  • Redundant Load/Store Elimination: Removing `STORE R0, a` followed immediately by `LOAD a, R0`.
  • Unreachable Code: Removing jumps to the next line.
  • Algebraic Simplifications: Replacing `ADD R0, #0` with nothing.
Unit 6 · 7 hrs
Advanced Compilation Topics
OOP compilation, Virtual Tables, Garbage Collection, and Tail-Call Optimization.

6.1 Object-Oriented Compilation

Modern OOP languages require dynamic dispatch for polymorphism.

  • Virtual Tables (v-tables): Every class with virtual methods gets an array of function pointers. Every object instance has a hidden pointer (`vptr`) pointing to its class's v-table. At runtime, the method is resolved via `obj->vptr[method_offset]()`.
  • Name Mangling: To support function overloading, compilers rename functions internally by appending type information to the function name (e.g., `_Z3addii` for `add(int, int)`).

6.2 Automated Memory Management

  • Reference Counting: Each object tracks how many pointers refer to it. Frees memory when count hits 0. Suffers from cyclic references.
  • Mark-and-Sweep: Halts execution, starts from root pointers, marks reachable objects, and sweeps away unmarked ones. Fixes cyclic references.

🎯 Supervised Learning
Linear / Logistic Regression
Definition:
• Linear Regression: Models a linear relationship between features and a continuous target.
• Logistic Regression: Predicts probability of a binary output squashed between 0 and 1.
Theory: Linear regression fits a straight line to minimize squared error. Logistic regression uses maximum likelihood estimation to fit a sigmoid probability curve.
Formula:
Linear: Y = β₀ + β₁X₁ + ... + βₙXₙ + ε Logistic: P(Y=1|X) = σ(z) = 1 / (1 + e⁻ᶻ) where z = β₀ + β₁X₁ + ... + βₙXₙ
Example: Predicting house prices based on size (Linear) or classifying an email as Spam/Not Spam (Logistic).
Support Vector Machines (SVM)
Definition: A classification/regression model that maps inputs into space and finds a boundary hyperplane with the maximum margin between classes.
Theory: Optimization of the hyperplane boundary parameters. Uses mathematical Kernel functions (the Kernel Trick) to project non-linear boundaries into higher dimensions.
Formula:
Hyperplane: wᵀx + b = 0 Optimization: min_w,b (1/2)||w||² s.t. yᵢ(wᵀxᵢ + b) ≥ 1 RBF Kernel: K(x_i, x_j) = exp(-γ||x_i - x_j||²)
Example: Image classification, facial recognition, or medical diagnostics.
Decision Trees
Definition: A non-parametric supervised model that splits dataset features recursively into subsets based on decisions to maximize information purity.
Theory: Build a tree structure where internal nodes test features and leaf nodes represent outcomes, selecting splits to reduce entropy.
Formula:
Entropy: H(S) = -Σ pᵢ log₂(pᵢ) Info Gain: IG(S, A) = H(S) - Σ (|S_v|/|S|) * H(S_v) Gini Impurity: Gini(S) = 1 - Σ pᵢ²
Example: Loan approval systems (If income > 50k -> If credit score > 700 -> Approve).

Neural Networks
Definition: Network of interconnected units (neurons) organized in layers, designed to model complex non-linear functions.
Theory: Learns mappings using forward propagation (features pass through weights and activations) and backpropagation (computes gradients of loss using Calculus chain rule to update weights).
Formula:
Neuron Output: a = g(z) = g(Σ wᵢxᵢ + b) ReLU Activation: g(x) = max(0, x) Weight Update: w ← w - η * (∂L/∂w)
Example: Handwritten digit recognition (MNIST), translation, object detection.
🔍 Unsupervised Learning
K-Means Clustering
Definition: An unsupervised clustering algorithm that partitions n observations into K distinct clusters where each observation belongs to the cluster with the nearest mean.
Theory: Iteratively assigns points to nearest centroids and updates centroids to the mean of their assigned points, minimizing within-cluster variance.
Formula:
Objective (WCSS): J = Σ_j=1ᴷ Σ_i∊S_j ||xᵢ - μ_j||² Centroid Update: μ_j = (1 / |S_j|) * Σ_i∊S_j xᵢ
Example: Customer segmentation based on purchasing behavior.
Hierarchical Clustering
Definition: A clustering method that builds a tree of clusters (dendrogram) either by recursively merging smaller clusters (Agglomerative) or splitting larger ones (Divisive).
Theory: Calculates distances between clusters using linkage criteria (Single, Complete, Average) to build hierarchical groupings without predefined cluster counts.
Formula:
Single Linkage: d(A, B) = min { d(x, y) : x∊A, y∊B } Complete Linkage: d(A, B) = max { d(x, y) : x∊A, y∊B } Average Linkage: d(A, B) = (1 / (|A|*|B|)) * Σ_x∊A Σ_y∊B d(x,y)
Example: Document taxonomy clustering, phylogenetic tree creation.
PCA (Dimensionality)
Definition: Principal Component Analysis is an unsupervised linear transformation technique used to project high-dimensional features into a lower-dimensional space.
Theory: Finds orthogonal axes (Principal Components) of maximum variance by calculating eigenvectors and eigenvalues of the data's covariance matrix.
Formula:
Covariance Matrix: Σ = (1 / n) * XᵀX Eigen Decomposition: Σv = λv where v = Principal Component, λ = Eigenvalue (Variance)
Example: Reducing 1000-dimensional images to 50 key features for facial recognition.

Definition: An ensemble method that builds weak decision trees sequentially, where each new tree is trained to predict the negative gradients (pseudo-residuals) of the loss function of the previous aggregate model.

Unit 1 · 8 hrs
Foundations of ML & Statistical Learning
Linear algebra, statistical learning theory, types of learning, hypothesis space, inductive bias, cross-validation, and optimization techniques.

1.1 Core Concepts

Machine Learning algorithms use a set of training data to search through a Hypothesis Space (H). The algorithm's preference for certain hypotheses over others (e.g., preferring simpler models) is called its Inductive Bias.

The Bias-Variance Tradeoff: The fundamental dilemma in ML.
- Bias: Error introduced by approximating a real-world problem with a simplified model. High bias leads to Underfitting.
- Variance: Error introduced by the model's sensitivity to small fluctuations in the training set. High variance leads to Overfitting.

Formula: Total Error = Bias² + Variance + Irreducible Noise

1.2 Model Evaluation

  • Holdout Method: Splitting data into Training (70%), Validation (15%), and Test (15%) sets.
  • k-Fold Cross Validation: Dataset is split into k equal partitions. Model trains on k-1 partitions and is tested on the remaining 1 partition. This rotates until every partition has been used exactly once for testing. Averages the results to prevent data leaking and overfitting.

1.3 Optimization (Gradient Descent)

To find the best model parameters `θ`, we minimize a Cost Function `J(θ)`. Gradient Descent updates parameters iteratively: θ = θ - α ∇J(θ), where `α` is the learning rate.

Unit 2 · 8 hrs
Bayesian Learning & Regression Models
Statistical decision theory, MAP/MLE estimation, Linear Regression, Regularization (Ridge, Lasso), PCA, Partial Least Squares.

2.1 Bayesian Learning

Definition: A statistical framework that updates the probability of a hypothesis based on observed data and prior distributions.

Formulas:

Bayes' Theorem: P(h|D) = [ P(D|h) * P(h) ] / P(D) MAP Estimation: h_MAP = argmax P(D|h) * P(h) MLE Estimation: h_MLE = argmax P(D|h)
  • Maximum A Posteriori (MAP): Estimates parameters by combining the likelihood of observed data with prior belief distribution.
  • Maximum Likelihood Estimation (MLE): Assumes uniform prior probabilities, optimizing parameter values to maximize likelihood of observed data.

2.2 Linear Regression & Regularization

Definition: Linear Regression models continuous variables via linear coefficients. Regularization prevents model overfitting by introducing parameter scale penalties.

Formulas:

OLS Objective: J(β) = (1/2n) * ||y - Xβ||² Ridge Penalty: J_ridge(β) = OLS_Cost + λ * ||β||₂² Lasso Penalty: J_lasso(β) = OLS_Cost + λ * ||β||₁

Linear Regression fits a model line `y = Xβ + ε`. When columns exhibit multicollinearity, regularized regression shifts weights closer to zero:

TypePenalty TermEffect on CoefficientsUse Case
Ridge (L2)+ λ∑β²Shrinks them towards zero (but never exactly zero).When dealing with highly correlated features (Multicollinearity).
Lasso (L1)+ λ∑|β|Can shrink coefficients exactly to zero.Feature Selection. Creates sparse models.
Elastic NetL1 + L2 combinedBalances Ridge and Lasso properties.When features outnumber samples.

2.3 Principal Component Analysis (PCA)

Definition: An unsupervised linear transform that projects features into a lower-dimensional coordinates system along axes of maximum variance.

Formulas:

Covariance Matrix: Σ = (1 / n) * XᵀX Eigen Decomposition: Σv = λv where v = Principal Component, λ = Eigenvalue (Variance)

PCA calculates the covariance matrix of variables, performing eigenvector analysis to drop components representing low variances.

Unit 3 · 9 hrs
Classification Algorithms & Neural Networks
Logistic regression, LDA, QDA, Perceptron, SVM (Kernel Tricks), Decision Trees (Information Gain), ANN + Backpropagation.

3.1 Linear Classifiers

  • Logistic Regression: Uses the Sigmoid function σ(z) = 1 / (1 + e^(-z)) to squash linear outputs into probabilities (0 to 1). Optimized via Maximum Likelihood Estimation.
  • LDA (Linear Discriminant Analysis): A generative model that assumes features follow a Gaussian distribution. Finds axes that maximize class separability while minimizing intra-class variance.

3.2 Support Vector Machines (SVM)

Max-Margin Hyperplane: SVM draws a decision boundary that maximizes the "margin" (distance to the nearest training points, called Support Vectors).

The Kernel Trick: When data is not linearly separable, SVMs project the data into a higher-dimensional space where a linear hyperplane can separate it. Instead of computing the mapping directly (which is computationally expensive), Kernels (like RBF, Polynomial, Sigmoid) compute the dot products in the higher dimension instantly.

3.3 Decision Trees

Builds a tree by splitting the dataset based on feature thresholds. The "best split" is chosen by maximizing Information Gain (derived from Entropy) or minimizing Gini Impurity.

3.4 Artificial Neural Networks (ANN)

Multi-layer perceptrons. Non-linearity is introduced via Activation Functions (ReLU, Tanh, Sigmoid).

Backpropagation Algorithm:
1. Forward Pass: Inputs pass through weights to generate an output. Loss is calculated.
2. Backward Pass: The Chain Rule from calculus is applied from the output layer back to the input layer to calculate the gradient of the loss function with respect to every weight.
3. Update: Weights are updated in the opposite direction of the gradient using Gradient Descent.

4.1 The Ensemble Philosophy

Ensembles combine the predictions of multiple "weak learners" (models that are only slightly better than random guessing) to create one highly accurate "strong learner".

Unit 5 · 5 hrs
Unsupervised Learning: Clustering
K-means, K-medoids, density-based (DBSCAN), hierarchical clustering, and evaluation metrics (Silhouette score).

5.1 Centroid & Density Based Clustering

  • K-Means: Iteratively assigns points to the nearest centroid, then updates the centroid to the mean of the points. Pros: Extremely fast. Cons: Requires `K` beforehand, assumes spherical clusters, highly sensitive to outliers.
  • DBSCAN: Groups points that are closely packed together based on an epsilon radius and a MinPts threshold. Points in low-density regions are flagged as Outliers. Pros: Finds arbitrarily shaped clusters, robust to noise.

5.2 Hierarchical Clustering

Builds a tree of clusters called a Dendrogram. Does not require specifying `K` in advance.

  • Agglomerative (Bottom-Up): Starts with each point as its own cluster, then merges the two closest clusters iteratively until only one root cluster remains.
  • Divisive (Top-Down): Starts with all points in one giant cluster, then splits them recursively.
Unit 6 · 7 hrs
Probabilistic Models & Reinforcement Learning
Gaussian Mixture Models (GMM), EM algorithm, Bayesian Networks, Markov Decision Processes (MDP), and Q-Learning.

6.1 Soft Clustering with GMMs

Gaussian Mixture Models (GMM): Assumes data is generated from a mixture of several Gaussian distributions. Unlike K-Means (hard clustering), GMM performs "soft clustering"—meaning a point belongs to a cluster with a specific probability.

The Expectation-Maximization (EM) Algorithm:
Used to find the maximum likelihood parameters for GMMs.
- E-Step: Given current parameters, estimate the probability that each point belongs to each cluster.
- M-Step: Given these probabilities, update the parameters (Mean, Variance, Weight) of the Gaussians to maximize the likelihood. Repeats until convergence.

6.2 Reinforcement Learning (RL)

In RL, an Agent interacts with an Environment. The agent observes a State (s), takes an Action (a), transitions to a new state `s'`, and receives a Reward (r). The goal is to learn an optimal Policy (π) that maximizes the cumulative discounted reward.

Q-Learning & The Bellman Equation:
A model-free RL algorithm. It updates an action-value function `Q(s,a)` iteratively:

Q(s, a) ← Q(s, a) + α [ r + γ max Q(s', a') - Q(s, a) ]

Where `α` is the learning rate, and `γ` is the discount factor for future rewards.

Machine Learning Comprehensive Glossary
Backpropagation: Algorithm for computing the gradient of the loss function with respect to neural network weights.
Bias-Variance Tradeoff: The balance between a model's ability to fit training data (low bias) and generalize to new data (low variance).
Cross-Entropy: Common loss function for classification tasks. Measures the difference between two probability distributions.
Epoch: One complete pass through the entire training dataset during the learning process.
Gradient Descent: Optimization algorithm used to minimize the loss function by iteratively moving in the direction of steepest descent.
Hyperparameter: A parameter whose value is set before the learning process begins (e.g., learning rate, number of layers).
Overfitting: When a model learns the detail and noise in the training data to the extent that it negatively impacts performance on new data.
Regularization: Techniques (like L1/L2 penalty) used to reduce overfitting by penalizing large weights.
Support Vector: The data points closest to the hyperplane in an SVM model. They dictate the position of the boundary.

105604 — Elective I
Graph Theory
Euler/Hamiltonian paths, trees, spanning trees, planar graphs, colouring, directed graphs.
  • Euler Path: A trail in a finite graph that visits every edge exactly once. It exists if and only if exactly 0 or 2 vertices have an odd degree.
  • Hamiltonian Path: A path in an undirected or directed graph that visits every vertex exactly once. Finding this is NP-complete.
  • Minimum Spanning Tree (MST): A subset of edges that connects all vertices with the minimum possible total edge weight. Solved greedily by Kruskal's or Prim's algorithms.

Planar Graphs & Coloring: A graph is planar if it can be drawn on a plane without edges crossing. The Four Color Theorem states that any planar graph can be colored with at most 4 colors such that no two adjacent vertices share the same color.

Unlike hard computing, soft computing is tolerant of imprecision, uncertainty, and partial truth.

Fuzzy Logic: Standard boolean logic is 0 (False) or 1 (True). Fuzzy logic allows truth values anywhere between 0 and 1. Used heavily in control systems (like AC temperature regulators).

Genetic Algorithms (GA): Inspired by Charles Darwin's theory of natural evolution. The algorithm repeatedly modifies a population of individual solutions. Phases include:
1. Selection: Selecting the fittest individuals.
2. Crossover: Combining two parents to create offspring.
3. Mutation: Randomly altering parts of the offspring to maintain genetic diversity.
